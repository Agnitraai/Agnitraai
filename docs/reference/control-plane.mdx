# API server reference

`agnitra-api` is the optional HTTP server that lets remote clients submit optimization jobs without running the SDK in-process. Browser and Node.js callers can use the [npm `agnitra` HTTP client](https://www.npmjs.com/package/agnitra).

## Endpoints

### `GET /health`

Liveness probe. Returns 200 when the server is accepting requests.

```bash
curl http://127.0.0.1:8080/health
# {"status": "ok"}
```

### `POST /optimize`

Submit an optimization job. The server queues the work and returns a job ID immediately.

```bash
curl -X POST http://127.0.0.1:8080/optimize \
  -H 'Content-Type: application/json' \
  -H "X-Agnitra-Api-Key: $AGNITRA_API_KEY" \
  -d '{
    "project_id": "my-project",
    "model_name": "meta-llama/Meta-Llama-3-8B-Instruct",
    "target": "H100",
    "tokens_processed": 4096
  }'
```

Response:

```json
{
  "job_id": "job_abc123",
  "status": "pending"
}
```

### `GET /jobs/{job_id}`

Poll for job status:

```bash
curl http://127.0.0.1:8080/jobs/job_abc123
```

Response:

```json
{
  "job_id": "job_abc123",
  "status": "completed",
  "result": {
    "baseline": { "latency_ms": 53.5, "tokens_per_sec": 53.5, ... },
    "optimized": { "latency_ms": 30.1, "tokens_per_sec": 75.0, ... },
    "detected_architecture": "llama",
    "notes": { ... }
  }
}
```

### `WebSocket /ws/jobs/{job_id}`

Real-time job status. Connect and receive a JSON `JobStatus` message every time the job state changes.

```javascript
const ws = new WebSocket('ws://127.0.0.1:8080/ws/jobs/job_abc123');
ws.onmessage = (ev) => console.log(JSON.parse(ev.data));
```

Or use the npm client's async iterator:

```typescript
for await (const update of client.subscribeToJob(jobId)) {
  console.log(update.status);
  if (update.status === "completed") break;
}
```

## Authentication

The server checks `X-Agnitra-Api-Key` against the `AGNITRA_API_KEY` environment variable when set. If `AGNITRA_API_KEY` is unset, all routes are open — that's only safe when the server binds to `127.0.0.1`. Always set an API key before binding to a public interface.

## Configuration

Set via environment variables:

| Variable | Default | Purpose |
|---|---|---|
| `AGNITRA_API_HOST` | `127.0.0.1` | Bind interface |
| `AGNITRA_API_PORT` | `8080` | Port |
| `AGNITRA_API_KEY` | unset | Required key for client requests |
| `AGNITRA_ALLOW_PUBLIC_BIND` | `0` | Set to `1` to silence the public-bind warning |
| `OPENAI_API_KEY` | unset | Enables LLM-guided optimization on the server |

## Running

```bash
# Local development
agnitra-api

# Production behind a reverse proxy
AGNITRA_API_HOST=0.0.0.0 \
AGNITRA_API_KEY="$(openssl rand -hex 32)" \
AGNITRA_ALLOW_PUBLIC_BIND=1 \
agnitra-api
```

For containerized deployment, use the root `Dockerfile`:

```bash
docker build -t agnitra-api .
docker run --rm -p 8080:8080 -e AGNITRA_API_KEY=secret agnitra-api
```
