# Quickstart

This page takes you from `pip install` to a measurably faster model in about five minutes.

## 1. Install

```bash
pip install agnitra
```

For real speedup against the HuggingFace baseline, install the quantization extra:

```bash
pip install "agnitra[quantize]"
```

Other extras:

| Extra | Adds | When you need it |
|---|---|---|
| `agnitra[quantize]` | `torchao` | INT8 weight-only quant ŌĆö the real speedup lever |
| `agnitra[openai]` | `openai`, `httpx` | LLM-guided optimization research path |
| `agnitra[rl]` | `stable-baselines3`, `gymnasium` | PPO-guided optimization research path |
| `agnitra[nvml]` | `pynvml` | GPU telemetry collection |

## 2. Optimize a HuggingFace model

The lowest-friction path uses the HuggingFace integration:

```python
import torch
from agnitra.integrations.huggingface import AgnitraModel
from transformers import AutoTokenizer

MODEL_ID = "meta-llama/Meta-Llama-3-8B-Instruct"

model = AgnitraModel.from_pretrained(
    MODEL_ID,
    torch_dtype=torch.float16,
    agnitra_kwargs={
        "input_shape": (1, 512),
        "quantize": "int8_weight",   # ŌåÉ the speedup lever
    },
).cuda()

tokenizer = AutoTokenizer.from_pretrained(MODEL_ID)
inputs = tokenizer("Explain backpropagation in one sentence:", return_tensors="pt").to(model.device)
print(tokenizer.decode(model.generate(**inputs, max_new_tokens=64)[0], skip_special_tokens=True))
```

`AgnitraModel.from_pretrained` is a drop-in replacement for `AutoModelForCausalLM.from_pretrained`. Pass any other `transformers.AutoModelFor...` class via `model_class=` for non-CausalLM workloads.

## 3. Or use the lower-level SDK

If you already have a model loaded, use `agnitra.optimize` directly:

```python
import agnitra

result = agnitra.optimize(
    model,
    input_shape=(1, 512),
    quantize="int8_weight",
)
fast_model = result.optimized_model
```

`result` is a [`RuntimeOptimizationResult`](/docs/reference/configuration#runtime-optimization-result) carrying the optimized model, baseline/optimized snapshots, and structured `notes` (validation drift, cache hits, fallback flags).

## 4. Verify

```python
print(result.notes)
# {'detected_architecture': 'llama', 'specialist': True, 'validation': {...}, ...}
```

If `result.notes["passthrough"] == True`, your model's architecture is outside Agnitra's [supported set](/docs/reference/configuration#supported-architectures) and the model came back unchanged. That's intentional ŌĆö Agnitra never silently no-ops.
