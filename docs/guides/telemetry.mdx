# Telemetry

Agnitra AI emits structured telemetry at every stage of the optimization pipeline so you can audit decisions, prove uplift, and feed downstream observability tools.

## What gets measured

| Stage | Captured | Where it lands |
|---|---|---|
| Detection | Architecture name (`model_type`), structural fallback flag | `result.notes["detected_architecture"]` |
| Fingerprint | Architecture + workload signatures | `result.notes["fingerprint"]`, `fingerprint_signature` |
| Specialist | Which architecture handler ran, quantize mode | `result.notes["specialist"]`, log line |
| Validation | Cosine similarity, max-abs-diff, argmax-match-rate | `result.notes["validation"]` |
| Cache | Hit / miss / store, TTL | `result.notes["cache_info"]` |
| Profiler | CUDA times, layer shapes, memory (when NVML available) | `OptimizationSnapshot.telemetry` |

## Reading telemetry from a result

```python
result = agnitra.optimize(model, input_shape=(1, 512), quantize="int8_weight")

print(result.notes["detected_architecture"])         # "llama"
print(result.notes["validation"]["cosine_similarity"])  # 1.0 - tolerance bound
print(result.baseline.latency_ms, result.optimized.latency_ms)
```

## Exporting to external systems

### Webhook notifications

Set `AGNITRA_NOTIFY_WEBHOOK_URL` (Slack, Discord, Telegram, or generic JSON) and Agnitra AI POSTs a summary to your webhook after every successful optimization:

```bash
export AGNITRA_NOTIFY_WEBHOOK_URL="https://hooks.slack.com/services/..."
```

Or pass a `WebhookNotifier` programmatically:

```python
from agnitra.core.notifications import WebhookNotifier

result = agnitra.optimize(
    model,
    input_shape=(1, 512),
    notify_webhook=WebhookNotifier.from_env(),
)
```

### Telemetry client (optional)

For high-volume deployments, attach a `TelemetryClient` that batches snapshots:

```python
from agnitra.core.runtime.telemetry_client import TelemetryClient, TelemetryConfig

client = TelemetryClient(TelemetryConfig(endpoint="https://your-collector.example.com"))
result = agnitra.optimize(model, input_shape=(1, 512), telemetry_client=client)
```

Set `offline=True` to disable network calls entirely (useful for air-gapped deployments and CI).

### NVML GPU metrics

Install the NVML extra to capture per-call GPU utilization, power draw, and memory pressure:

```bash
pip install "agnitra[nvml]"
```

Snapshots include the `gpu_utilization` field automatically when NVML is present.

## What Agnitra AI does NOT collect

Agnitra AI is a local SDK by default. Without explicit configuration:

- No data is sent to any Agnitra AI-operated service.
- No model weights, input tensors, or generated outputs are persisted.
- Architecture fingerprints stay in-process.

The only exception is the LLM-guided research path (`enable_rl=True` with `OPENAI_API_KEY` set), which sends graph IR to OpenAI for kernel suggestions. The specialist path used by `use_specialist=True` (the default) makes no network calls.

See [configuration reference](/docs/reference/configuration#environment-variables) for every environment variable that affects telemetry behavior.
