# vLLM is intentionally NOT in this requirements file.
#
# vllm 0.5.4 has a transitive dependency on pyairports via outlines.
# Only pyairports 0.0.1 exists on PyPI, and that version has a
# packaging issue that leaves the module installed-but-unimportable
# inside vllm's process. This blocks the entire image build.
#
# The vllm runner (runners/vllm_runner.py) gracefully skips when
# vllm is not installed — it writes a JSON result with success=false
# and library_version="vllm (not installed)". The vLLM column in
# RESULTS.md is therefore left blank for this benchmark suite until
# the dep mess is solved properly (likely by upgrading torch + vllm
# together to a newer pair).
#
# Re-add `vllm==<pinned-version>` here when ready and rebuild the
# Modal image. Track in: TODO(benchmark): restore vllm column.
